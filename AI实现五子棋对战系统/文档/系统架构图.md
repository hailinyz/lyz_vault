## 2. 系统架构说明

### 2.1 架构层次

#### 2.1.1 用户层 (User Layer)

- **Web浏览器客户端**：基于HTML/CSS/JavaScript原生技术栈开发，支持WebSocket实时通信，提供五子棋游戏界面和用户交互功能

#### 2.1.2 网络通信层 (Network Layer)

- **HTTP RESTful API**：处理用户注册、登录、查询等标准HTTP请求
- **WebSocket实时通信**：支持游戏对战过程中的实时消息推送，包括落子同步、游戏状态更新、匹配状态通知

#### 2.1.3 应用层 (Application Layer)

- **Spring Boot 3.x框架**：基于JDK 17运行，内置Tomcat服务器在8080端口提供Web服务
- **业务服务层**：包含六大核心业务模块，负责处理五子棋平台的所有业务逻辑

#### 2.1.4 数据访问层 (Data Access Layer)

- **MyBatis-Plus ORM**：提供对象关系映射，简化数据库操作
- **JWT认证框架**：处理用户身份认证和会话管理
- **SLF4J + Logback**：提供系统日志记录和管理

#### 2.1.5 基础设施层 (Infrastructure Layer)

- **MySQL 8.0+数据库**：存储用户信息、游戏记录、积分流水等持久化数据
- **Redis 6.0+缓存**：提供高性能缓存服务，存储会话信息和临时数据
- **RabbitMQ 3.8+消息队列**：处理异步消息传递和系统解耦
- **Maven构建工具**：管理项目依赖和构建流程

### 2.2 业务模块架构

#### 2.2.1 用户管理模块 (User Module)

- **职责**：负责用户基础信息管理，包括用户注册、信息查询更新、游戏统计数据维护
- **核心功能**：用户注册验证、用户CRUD操作、初始积分和统计数据创建

#### 2.2.2 认证授权模块 (Auth Module)

- **职责**：专门处理用户身份认证和会话管理，确保系统安全性
- **核心功能**：登录验证、JWT Token生成、会话状态维护、登录错误处理
- **技术实现**：JWT令牌机制、密码加密验证

#### 2.2.3 积分管理模块 (Score Module)

- **职责**：处理积分系统的所有业务逻辑，确保积分计算准确性
- **核心功能**：积分计算(胜利+5分，失败-3分，最低0分)、积分流水记录、历史查询

#### 2.2.4 匹配系统模块 (Matching Module)

- **职责**：提供公平高效的在线匹配服务
- **核心功能**：匹配队列管理、基于积分的匹配算法、匹配状态实时更新、房间分配
- **算法策略**：积分就近匹配保证公平性、黑白方随机分配

#### 2.2.5 游戏核心模块 (Game Module)

- **职责**：处理五子棋游戏的核心业务逻辑和规则
- **核心功能**：游戏房间管理、15×15棋盘状态维护、落子验证、五子连珠判定、回合控制
- **游戏规则**：黑方先行、轮流落子、横竖斜五子连珠判胜、认输处理

#### 2.2.6 实时通信模块 (Communication Module)

- **职责**：提供客户端与服务器间的实时通信能力
- **核心功能**：WebSocket连接管理、实时消息推送、连接状态监控、房间信息同步
- **消息类型**：落子同步、游戏状态变化、匹配状态更新、积分变化通知

### 2.3 模块依赖关系

```
认证授权模块 ← 所有业务模块（提供安全验证）
    ↑
用户管理模块 ← 认证授权模块、积分管理模块、匹配系统模块
    ↑
积分管理模块 ← 游戏核心模块（接收胜负结果）
    ↑              ↓（提供积分数据）
匹配系统模块 → 游戏核心模块（成功匹配后通知）
    ↓
实时通信模块 ← 匹配系统模块、游戏核心模块（提供消息传递）
```

### 2.4 安全架构

#### 2.4.1 身份认证安全

- **JWT令牌机制**：采用JSON Web Token进行无状态身份认证，避免服务端会话存储
- **密码安全**：用户密码采用加密存储，登录时进行安全验证
- **会话管理**：通过Redis缓存管理用户会话状态，支持会话过期和刷新


#### 2.4.2 通信安全

- **API安全**：RESTful API采用JWT认证，防止未授权访问

### 2.5 性能架构

#### 2.5.1 缓存策略

- **Redis缓存**：用户会话信息、频繁查询的数据存储在Redis中，提高响应速度
- **数据库连接池**：使用连接池管理数据库连接，避免频繁创建连接的性能损耗

#### 2.5.2 负载均衡

- **内置Tomcat**：Spring Boot内置Tomcat服务器，支持高并发请求处理
- **消息队列**：RabbitMQ处理异步消息，避免同步处理造成的性能瓶颈

#### 2.5.3 数据库优化

- **MyBatis-Plus**：提供高效的ORM映射，支持分页查询和条件构造
- **数据库索引**：针对查询频繁的字段建立索引，提高查询性能

## 3. 部署架构

### 3.1 开发环境部署

- **应用服务**：Spring Boot应用运行在8080端口
- **数据库服务**：MySQL运行在3306端口  
- **缓存服务**：Redis运行在6379端口
- **消息队列**：RabbitMQ运行在5672端口，管理界面15672端口
- **前端资源**：通过Spring Boot内置服务器提供静态资源服务

### 3.2 生产环境部署

- **操作系统**：Linux (Ubuntu 22.04)
- **Java运行环境**：JRE 17
